# 项目介绍

[English](../en-US/PROJECT_INTRODUCTION.md) | [简体中文](PROJECT_INTRODUCTION_zh-CN.md)

## 项目是什么

**Minecraft Earth Map for NovoAtlas**，项目品牌名为 **TerraScale**，是一个面向 Minecraft 1.21.1 / NeoForge 的真实数据全球地球地图生成项目。它的目标不只是复刻地球轮廓，而是让玩家能够通过地形辨认现实地理：山系来自高程数据，气候和土地覆盖决定地表群系，河流与海洋数据塑造水域环境，可选矿产层则把富矿区域放到现实主要矿区附近。

项目将这些数据转换为修改版 NovoAtlas 可以读取的瓦片化高度图、地表群系图和地下矿产图。太平洋中心坐标布局让主要大陆尽量完整地落在常用探索区域；可断点续跑的 432 地理分区流水线，则避免用一张低分辨率全球图粗略放大各地细节。

TerraScale 想同时做到三件通常互相制约的事：让整个地球一眼可辨、让局部区域值得探索，并让如此巨大的地图仍能被 Minecraft 服务器实际加载和逐区生成。

项目提供两个可以直接创建世界的数据包比例：

- **1:200**：一格水平方向约代表 200 米，适合追求更高海岸与区域细节的世界。
- **1:400**：一格水平方向约代表 400 米，世界跨度和服务器生成成本更低。

地图使用 Pacific-centered 布局，把全球地图接缝放在 `31.5°W`，地图中央约为 `148.5°E`。这样亚洲、澳洲、太平洋和美洲可以以经典太平洋中心世界地图的方式摆放，大陆不会在常用活动区域被随意切开。

## 为什么修改 NovoAtlas

原始 NovoAtlas 使用普通图片描述地图。全球 1:200 或 1:400 图层如果保存为单张超大 PNG，会带来数据包验证失败、图片解码内存峰值、启动卡住和修改困难等问题。

本项目修改 NovoAtlas，使其支持 tiled image map：

```text
height/{tx}_{tz}.png
surface/{tx}_{tz}.png
ore/{tx}_{tz}.png
```

每张瓦片为 2048 x 2048。NovoAtlas 只读取需要的瓦片，构建程序也可以单独替换某个地理区域，而不必重新保存一张全球超大图片。

## 最终数据包如何工作

最终服务器只加载一个普通数据包。432 个地理分片只存在于离线构建过程：

1. 脚本按经纬度范围生成一个分片；
2. 分片使用全球绝对坐标采样数据；
3. 高度和群系结果写入全球瓦片中的对应位置；
4. 所有分片完成后，服务器加载拼装后的完整地图包。

它不是 432 个数据包同时管理不同区域，也不是预生成好的 Minecraft 存档。玩家进入未生成区块时，NovoAtlas 根据数据包实时生成对应地形。

## 两套矿产版本

每个比例提供两种地下规则：

- **Original Rich Ore**：保留项目早期整理的富矿分布。
- **Top100 Rich Ore**：按矿种整理全球主要矿区，分为五级富庶程度。

当前富矿通过自定义 cave biome 实现，可以按全球坐标控制矿区，但可能覆盖原地下群系并影响古城、繁茂洞穴和紫水晶晶洞。只需要地球地表的用户可以移除整个自定义 Ore 图层。

## 适合哪些用户

- 希望运营地球比例生存、国家经营、交通或航空服务器的管理者；
- 需要真实大陆、海洋、气候和矿产空间关系的整合包作者；
- 希望研究 NovoAtlas、栅格地理数据和 Minecraft 世界生成的开发者；
- 需要在 1:100、1:300、1:800 等新比例上继续开发的人。

## 项目不包含什么

- 不自动生成真实城市建筑、道路、铁路或国界；
- 不提供已经预生成全部区块的世界存档；
- 不保证现实测绘精度或最新矿业状态；
- 不保证所有模组结构与自定义 cave biome 完全兼容。

项目的目标是在现实地理辨识度、Minecraft 可玩性、数据包规模和服务器性能之间取得可复现的工程平衡。
