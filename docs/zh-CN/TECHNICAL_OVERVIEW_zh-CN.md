# 技术概览

## 架构

TerraScale 将全球数据拆分为三个独立图层：

1. **Height**：决定海底、海平面和陆地高程。
2. **Surface**：将地表颜色映射到 Minecraft 群系。
3. **Ore**：通过地下 cave biome 映射控制富矿组合。

修改版 NovoAtlas 不再要求每层使用单张超大图片，而是按 `2048 x 2048` 像素读取瓦片。这样可以降低数据包验证和运行时加载超大 PNG 时的内存压力，也允许构建程序只替换受影响的区域。

## 比例

| 参数 | 1:200 | 1:400 |
|---|---:|---:|
| 水平比例 | 1 block 约 200 m | 1 block 约 400 m |
| 全球宽度 | 163840 blocks | 81920 blocks |
| 全球高度 | 92160 blocks | 46080 blocks |
| 地理分片 | 432 | 432 |
| 标准分片 | 13.5° x 12° | 13.5° x 12° |
| 标准分片像素 | 6144 x 6144 | 3072 x 3072 |
| NovoAtlas 瓦片 | 2048 x 2048 | 2048 x 2048 |

水平比例不等于垂直比例。真实高程会压缩到 Minecraft 可用高度范围，保留海平面、平原、高原和高山的相对关系，而不会把 8848 米直接变成 8848 格。

## 地理分片

所有分片使用相同的全球经纬度规则和绝对坐标。构建程序按经纬度范围采样地理数据，生成临时高度与群系图，再将结果写入全球瓦片中的对应像素位置。

分片只用于离线构建。最终服务器加载的是一个完整数据包，不需要同时运行 432 个分片包，也不存在多个数据包各自管理一块世界的问题。

## 地表与群系

地表分类综合使用：

- ESA WorldCover 土地覆盖；
- WorldClim 温度与降水；
- 高程和坡度；
- 纬度、海陆位置和海洋温度；
- HydroRIVERS 与 Natural Earth 水系。

海洋按温度与深度映射为暖水、温水、冷水、冻结及深海变体。陆地覆盖进一步映射为平原、森林、热带、稀树草原、荒漠、湿地、积雪和山地群系。

1:400 版会根据分片位置选择相交的 HydroRIVERS 大陆档案，因此不再把亚洲河流数据错误地当作全球数据。南纬 60 度以南保留真实海岸和高程，陆地最终映射为雪原、雪坡或冰封山峰。

## 富矿层

Ore 图层不是直接放置方块，而是选择地下 cave biome。每一种 cave biome 包含对应的 configured features 和 placed features，从而生成特定矿物及等级。

Top100 版按矿种整理主要矿区，并使用五级富庶程度控制影响范围和生成密度。矿区重叠时使用组合编码，使同一区域可以同时包含多种富矿。

## 构建工具

`tools/1_200` 与 `tools/1_400` 包含当前生产管线，`tools/pipeline` 包含两种比例共享的基础真实数据分类逻辑。工具支持：

- 地理分片生成；
- 全球队列与断点续跑；
- 瓦片拼装；
- Top100 富矿层生成；
- 最终数据包打包。

公开脚本通过环境变量指定输出、缓存和源数据位置，已移除原构建电脑的个人绝对路径。运行这些脚本不是安装地图的必要步骤。
