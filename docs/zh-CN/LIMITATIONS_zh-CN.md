# 已知不足与适用范围

本项目已经可以用于正常创建和运行全球地球地图世界，但仍有以下限制。发布者和服务器管理者应在下载页面中保留本说明，不应将项目描述为完全精确的现实世界复刻。

## 地理与群系

- **1:200 水系不完整**：1:200 版本沿用较早的亚洲 HydroRIVERS 管线，亚洲以外部分区域可能没有应有的河流。湖泊、海岸和海洋不等同于河流层，因此仍可能正常显示。
- **部分高地分类偏差**：安第斯、落基山及青藏高原边缘等寒冷或干旱高地，可能被映射为偏橙色的干旱山地或恶地类群系。该现象通常是群系分类偏差，不代表数据包主动生成大面积岩浆。
- **地形是压缩模型**：水平比例为 1:200 或 1:400，垂直高度则压缩到 Minecraft 可用范围，两者不是相同比例。
- **细小地物有限**：狭窄河道、小岛、城市湖泊和细碎海岸可能受源数据分辨率与栅格化影响而变宽、消失或简化。
- **极区采用保护规则**：1:400 南极陆地会在真实海岸和高程基础上强制使用积雪群系，以避免缺测数据造成绿色陆地。这是有意的稳定性取舍。

## 世界内容

- 本项目不生成真实城市建筑、道路、铁路、行政边界或人口分布。
- Minecraft 结构仍由游戏和其他模组按照各自规则生成，可能与现实地点无关。
- 已生成区块不会随数据包升级而自动改变。修复只会影响之后生成的区块，除非删除旧区块或建立新世界。

## 富矿层与地下结构

当前 Original 和 Top100 富矿都通过 NovoAtlas cave-biome 图层工作：矿区颜色会把地下群系替换为 `world:ore_combo_*` 自定义群系。它不是单纯向原群系追加 placed feature。

因此富矿覆盖区域可能出现：

- 深暗之域被替换，依赖 `deep_dark` 或对应 biome tag 的古城无法生成；
- 繁茂洞穴、溶洞等原洞穴群系及其植被、装饰减少或消失；
- 紫水晶晶洞等没有复制进 `ore_combo_*` feature 列表的地下特征不再生成；
- 其他模组按 biome tag 添加的洞穴结构、怪物或装饰无法识别自定义矿区群系。

影响只发生在自定义 Ore 图层实际覆盖的地下区域，并非全球所有地下都必然缺失这些内容。但矿区越大、重叠越多，受影响面积越大。

需要优先保持原版地下内容时，应使用 `tools/ore/make_no_custom_ore_variant.py` 移除 cave-biome 富矿层。更理想的长期实现是修改模组，使富矿作为坐标感知的 placed feature 注入原群系，而不是替换原 cave biome；仅给自定义群系补几个 feature 不能完整恢复深暗之域、繁茂洞穴和结构标签语义。

## 性能与存储

- 数据包采用瓦片读取以降低超大单图内存压力，但首次生成新区块仍明显慢于普通原版世界。
- 使用 Chunky 预生成完整全球地图会消耗大量 CPU 时间和磁盘空间；大多数服务器更适合只预生成出生区域和活动范围。
- BlueMap 等网页地图的渲染缓存可能远大于数据包本身，不属于本项目数据包体积。

## 模组兼容性

- 当前四个 Release 数据包需要 Create，因为富矿层包含 Create 锌矿注册项。
- 其他修改生物群系、噪声路由、矿物 placed feature 或维度生成器的模组可能覆盖或叠加本项目规则。
- 添加新矿石时，客户端与服务端都必须安装提供对应方块注册项的模组。

## 数据与精度

- 不同地理数据集的年份、分辨率和边界定义并不完全相同，叠加后可能出现局部错位。
- 富矿等级是面向游戏玩法的空间模型，不等于现实储量证明、矿权范围或经济评价。
- 本项目不应用于导航、测绘、工程、安全或投资决策。
