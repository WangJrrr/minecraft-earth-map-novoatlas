# Rebuilding the 1:200 and 1:400 Editions

[English](REBUILD_GUIDE.md) | [简体中文](../zh-CN/REBUILD_GUIDE_zh-CN.md)

This guide is for developers changing terrain, hydrology, scale, or ore rules. Players installing a release do not need the build pipeline.

## Environment

Use 64-bit Python 3.11+ and sufficient disk space.

```powershell
cd C:\path\to\NovoAtlas-EarthMap
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Set build locations:

```powershell
$env:NOVOATLAS_OUTPUT_ROOT = "D:\NovoAtlasBuild\outputs"
$env:NOVOATLAS_CACHE_ROOT = "D:\NovoAtlasBuild\cache"
$env:NOVOATLAS_PIPELINE_ROOT = "$PWD\tools\pipeline"
$env:NOVOATLAS_MINERAL_INPUT_ROOT = "D:\NovoAtlasBuild\inputs\mineral_data"
```

Environment variables apply to the current PowerShell session.

## Geographic Data

Prepare these under the cache root:

```text
wc2.1_2.5m_tavg.zip
wc2.1_2.5m_prec.zip
terrarium_tiles/
worldcover_2021_z8/
```

WorldCover and some elevation tiles can be downloaded on demand. WorldClim archives must be obtained beforehand.

Prepare under `tools/pipeline/vector_cache`:

```text
HydroRIVERS_v10_as_shp.zip
ne_10m_lakes.zip
```

The fixed 1:400 `global_hydrorivers.py` downloads additional continental HydroRIVERS archives when required. Follow each provider's license and attribution rules.

## Use a Release Pack as a Template

The build queue needs metadata, dimensions, surface-biome definitions, and worldgen files. The simplest reproducible template is an unpacked release pack of the desired scale. Height and surface regions are still regenerated by the queue.

For a no-Create or no-rich-ore build, convert the template first with `make_no_custom_ore_variant.py`.

## Rebuild 1:200

Copy an unpacked 1:200 template to:

```text
%NOVOATLAS_OUTPUT_ROOT%\NovoAtlas_World_1block200m_Global_HQ_Pacific_Assembly
```

Remove an old queue state if a completely fresh run is intended:

```powershell
Remove-Item "$env:NOVOATLAS_OUTPUT_ROOT\world_1_200_global_hq_pacific_region_queue_state.json" -ErrorAction SilentlyContinue
Remove-Item "$env:NOVOATLAS_OUTPUT_ROOT\world_1_200_global_hq_pacific_region_queue.log" -ErrorAction SilentlyContinue
```

Run:

```powershell
python tools\1_200\run_global_hq_pacific_region_queue.py
```

The queue calls the region builder and patches each result into the Pacific-centered assembly. Re-running the same command skips regions already recorded as complete.

Progress:

```powershell
Get-Content "$env:NOVOATLAS_OUTPUT_ROOT\world_1_200_global_hq_pacific_region_queue_state.json"
Get-Content "$env:NOVOATLAS_OUTPUT_ROOT\world_1_200_global_hq_pacific_region_queue.log" -Tail 20
```

The current public 1:200 builder retains the older Asia-focused HydroRIVERS path. A fully global-river 1:200 fork should integrate `tools/1_400/global_hydrorivers.py` and use a new river-mask cache name.

## Rebuild 1:400

Copy the unpacked Original template to:

```text
%NOVOATLAS_OUTPUT_ROOT%\FINAL_PACKS_1_400_Pacific\OriginalRichOre_1_400_Pacific_Tiled
```

For automatic Top100 finalization, also place the Top100 template beside it as:

```text
Top100RichOre_1_400_Pacific_Tiled
```

Set a dedicated work root and initialize:

```powershell
$env:NOVOATLAS_1_400_HQ_ROOT = "D:\NovoAtlasBuild\outputs\HQ_1_400_Rebuild"
python tools\1_400\initialize_global_hq_pacific_assembly_1_400.py
```

Start the queue:

```powershell
python tools\1_400\run_global_hq_pacific_region_queue_1_400.py
```

Optionally start the finalization watcher in another terminal:

```powershell
$env:NOVOATLAS_OUTPUT_ROOT = "D:\NovoAtlasBuild\outputs"
$env:NOVOATLAS_1_400_HQ_ROOT = "D:\NovoAtlasBuild\outputs\HQ_1_400_Rebuild"
python tools\1_400\watch_and_finalize_global_hq_pacific_1_400.py
```

Do not run the automatic Top100 finalizer for a no-custom-ore build. The queue's completed Original assembly can instead be processed with the no-custom-ore tool.

Progress:

```powershell
Get-Content "$env:NOVOATLAS_1_400_HQ_ROOT\world_1_400_global_hq_pacific_region_queue_state.json"
Get-Content "$env:NOVOATLAS_1_400_HQ_ROOT\world_1_400_global_hq_pacific_region_queue.log" -Tail 20
```

## Stop and Resume

- `Ctrl+C` safely stops a foreground queue. The active region may need to be rebuilt.
- Do not force power loss while PNG or assembly tiles are being written.
- Resume with the same output, cache, and state paths.
- After changing algorithms, use a new cache namespace or remove the affected cache so old masks do not hide new behavior.

## Validation Before Deployment

1. Confirm `pack.mcmeta` matches Minecraft 1.21.1.
2. Confirm map-info dimensions and tile paths match generated files.
3. Confirm height, surface, and Ore tile indices cover the complete grid.
4. Create a new test world and pass datapack validation.
5. Inspect coast, river, Antarctic, mountain, seam, and ore locations.
6. Search mod namespaces and install every referenced content mod on both sides.
